# Meeting AI — Anna Hackathon Submission

**One line:** Meeting AI turns a meeting transcript (typed, pasted, or spoken) into a
clean recap with action items, owners, and next steps — running entirely on Anna's
host LLM (`anna.llm.complete`), **no API key, no backend**.

Repo: https://github.com/thavionai/meeting-ai-anna

---

## The workflow (one focused meeting flow)

| Step | What the app does | Where |
|---|---|---|
| 1. **Input** | Take a short transcript — paste it, or click 🎤 **Listen** to capture live notes (browser Web Speech, key-free) | transcript box |
| 2. **Clean recap** | Click **Recap** → a 2–4 sentence summary of the meeting | recap card |
| 3. **Action items + owners** | The recap lists each action as **"Owner — task"** | recap card |
| 4. **Follow-up questions / next steps** | The recap suggests 2–4 next steps; live Q&A also answers questions as they come up | recap card / Q&A cards |
| 5. **Review, then use/share** | Everything is shown in-app for review first; then **Share** or post to the **Anna chat**, with **History** to revisit | Share / Export / History |

All reasoning runs through **`anna.llm.complete()`** — the model, quota, and billing
stay on the Anna Host.

---

## Why it's a good Anna App

- **Keyless AI:** every recap/answer is `anna.llm.complete(...)` — no OpenAI/Anthropic
  key in the app. The host provides the model.
- **Anna-native persistence:** history is stored with `anna.storage`; export posts the
  recap into the conversation with `anna.chat.append_artifact`.
- **Real product:** Meeting AI also ships as a desktop app (thavionai.com); this is the
  same workflow as a native Anna App bundle.

## Run it (step by step)

> Prereqs: `anna-app` CLI installed and logged in (`anna-app login`); Node ≥ 18.

```bash
git clone https://github.com/thavionai/meeting-ai-anna
cd meeting-ai-anna
anna-app validate      # ✓ validate passed
anna-app dev           # ✓ dashboard http://localhost:5180/
```

Open **http://localhost:5180/**, launch the **Meeting AI** window. The status pill
should read **“Connected · anna.llm.complete()”**.

## Demo script (≈2 minutes)

1. **Show it's keyless.** Point at the status pill: *“This app calls Anna's host LLM —
   there's no API key anywhere in it.”*
2. **Input a transcript.** Paste the sample below (or click 🎤 **Listen** and read a
   couple of lines out loud).
3. **(Optional wow) Live Q&A.** While listening, ask *“What time is the submission
   deadline?”* — an answer card appears, generated by `anna.llm.complete`.
4. **Recap.** Click **Recap**. Show the output: **Recap → Action items (with owners) →
   Follow-up questions / next steps → Decisions**.
5. **Review.** *“The user sees everything before sharing.”*
6. **Share / save.** Click **Share** (copies/shares the recap) or **.txt/.doc/PDF**
   (posts the recap into the Anna chat as an artifact). Open **History** to show it's
   saved and filterable by day.

## Anna features used

- `anna.llm.complete` — recap, live answers (host LLM, keyless)
- `anna.storage` — persistent history (per-app)
- `anna.chat.append_artifact` — share/export the recap into the conversation
- `anna.window.resize` — minimize to a compact bar
- Manifest: schema-2 UI bundle + `ui.host_api` grants (`llm`, `storage`, `chat`, `window`, `upload`)

## Honest notes (platform boundaries)

- **File downloads** (real `.pdf/.docx`) need an account-level **`upload_grant`**; without
  it, export posts the recap to the **Anna chat** instead (working fallback).
- **Stealth mode** and **system-audio capture** are desktop-only (Electron/OS) and don't
  exist in a sandboxed web app — they remain in the desktop Meeting AI.
- Live mic uses the browser **Web Speech API**; it needs mic permission in the app window.
